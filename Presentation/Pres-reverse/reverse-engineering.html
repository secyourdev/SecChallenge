Concepts clés avant de se lancer dans le reverse engineering

Tout programme executé dans un ordinateur doit être compiler dans un binaire pour être lisible par la machine.

Il est donc indispensable quand on fait du Reverse engineering et quand souvent le code source d'un programme n'est pas disponible

d'aller étudier le code assembleur, qui lui est facilement retrouvable depuis le binaire.

Fonctionnement d'un ordinateur

Un ordinateur se compose:

-D'un processeur contenant des registres permettent de stocker des données et de les manipuler extrêmement rapidement.

Lui servant à exécuter des instructions sur ces registres (additions, multiplications, comparaisons ...).

Le processeur peut aussi écrire dans la RAM (instruction load et store).

- La RAM, à beauoup plus de capacité de stockage comparativement au processeur. Elle a pour rôle de stocker les informations

et de fournir un espace de travail pour le processeur.

- Les périphériques externes( HDD,SSD, GPU, souris, clavier, etc ...)

Rôle du compilateur

La plupart des compilateurs visent la production de code assembleur le plus performant possible.

D'ailleurs avec l'utilisation d'un décompilateur il est très facile de retrouver le code assembleur et de pouvoir faire des premières analyses.

Cependant les compilateurs peuvent opérer de très grand changement à la structure du programme entre le C et l’assembleur pour améliorer les temps d’exécution.

Ce qui rend la convertion de l'assembleur au language C complexe. Malgré tous la NSA à développer un ghidra convertant le code assembleur en code C.

L'architecture x86

L'assembleur est un langage de programmation de bas niveau qui est utilisé pour écrire des programmes directement en langage machine pour les processeurs x86.

L'architecture x86 est largement utiliser dans les PC et les serveurs. On la retrouve essentielement dans les processeurs Intel et AMD.

Même si le l'architecture x86 est limité à des registres de 32 bits. Il y a de forte chance que votre ordinateur puisse l'exectuté même si il est en 64bits.

Ce qui dans la perceptive d'un hacker se réveler avantageux car il n'a qu'à écrire un programme en x86 pour pouvoir s'executer dans les architecture 32 bits et 64 bits.

D'un point de vue plus technique les registres des processeur x64 pour 64 bits ne sont utilisables que en x64 et pas en x86.

Les autres registres sont utilisables dans les deux modes.

Les flags

Les "flags" en assembleur sont des indicateurs d'état du processeur utilisés pour effectuer des tests et des sauts conditionnels.

Les flags peuvent être utilisés pour déterminer si une instruction doit être exécutée ou non en fonction de l'état du processeur.

Les flags les plus couramment utilisés incluent le "zero flag", le "carry flag" et le "sign flag".
