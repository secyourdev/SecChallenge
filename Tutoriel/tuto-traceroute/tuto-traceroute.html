Traceroute

1. Objectif

L'objectif de ce tutoriel est de vous présenter l'outil de ligne de commande traceroute.
traceroute relève le chemin emprunté par les paquets IP à travers un réseau. Cela renvoie les sauts successifs effectués par les paquets depuis la source vers la destination, avec les adresses IP des passerelles ainsi que les temps de réponse.
C'est un outil précieux en cybersécurité pour comprendre la topologie d'un réseau, détecter les problèmes de connectivité, identifier les anomalies ou les tentatives d'interception.

2. Configuration

Pour ce tutoriel nous allons utiliser trois machines (Alice, Eve et Bob) interconnectées via deux réseaux distincts en utilisant Docker et docker-compose. Téléchargez le docker-compose.yml
Voici les commandes pour démarrer les trois machines :
bash> docker-compose up -d

Pour accéder à la machine d'Alice, utilisez la commande suivante.
bash> docker exec -it --privileged reseau1_alice /bin/bash

On installe les paquets utiles pour la suite bash> apt-get install net-tools iputils-ping traceroute

On doit maintenant définir les routes de routage. Attention a bien les parametrer sur les deux machines alice et bob.
Dans le cas d'Alice par exemple : bash> route add default gw 192.168.111.3 eth0

2. Utilisation

On ping les machines pour savoir si elles sont atteignables

On effectue un traceroute sur la machine bob.

On observe que eve est aussi sur le réseau 2 où bob est présent et sert de routeur.
