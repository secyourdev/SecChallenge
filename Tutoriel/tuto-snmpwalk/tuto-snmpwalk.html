SnmpWalk

1. Objectif

L'objectif de ce tutoriel est de vous présenter l'outil de ligne de commande SnmpWalk.
SnmpWalk est un outil utilisé pour interroger et parcourir une base d'informations de gestion SNMP (MIB).
La MIB est une base de données hiérarchique qui stocke les informations gérées par un appareil SNMP.
SNMP (Simple Network Management Protocol) est un protocole très utilisé pour la gestion et la surveillance des dispositifs de réseau comme les routeurs, les commutateurs et les serveurs.
SNMP permet de récupérer les informations sur l'état, les performances et la configuration de ces appareils.

2. Configuration

Pour ce tutoriel nous allons utiliser trois machines (Alice, Eve et Bob) interconnectées via deux réseaux distincts en utilisant Docker et docker-compose. Téléchargez le docker-compose.yml
Voici les commandes pour démarrer les trois machines :
bash> docker-compose up -d

Pour accéder à la machine Alice, utilisez la commande suivante :
bash> docker exec -it --privileged reseau1_alice /bin/bash

Installation des paquets utiles pour la suite :
bash> apt-get update && apt-get install snmpd net-tools iputils-ping nano

On doit maintenant définir les routes de routage. Attention a bien les paramétrer sur les deux machines Alice et Bob.
Dans le cas d'Alice par exemple :
bash> route add default gw 192.168.111.3 eth0

3. Utilisation


bash> snmpwalk -v 2c -c public 192.168.112.2

4. Application

Ouvrir deux fenêtres et se connecter respectivement à Bob et à Alice.
Depuis le conteneur Alice nous allons lancer une capture.
bash_alice> docker exec -it reseau1_alice tcpdump -i eth0 -w capture.pcap
