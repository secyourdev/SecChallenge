Tutoriel : The Sleuth Kit

1. Objectif

L’objectif de ce tutoriel est de vous présenter le logiciel TheSleuthKit.
The Sleuth Kit est une collection de logicielle utilisés dans le domaine de la cybercriminalité pour effectuer l’analyse et la récupération de données sur les systèmes de fichiers. Il est utilisé par les professionnels de la sécurité, les chercheurs en informatique forensique, les forces de l'ordre et d'autres experts pour examiner les systèmes informatiques compromis et recueillir des preuves numériques.

2. Configuration

Pour ce tutoriel, nous utiliserons docker pour avoir une machine configurée avec The Sleuth Kit et divers fichiers sur lesquelles tester le logiciel.

Récupérer le dockerfile dans le lien ci-après : docker-compose.yml
Dans un bash (linux) ou cmd (windows) aller dans le répertoire où vous avez télécharger le fichier docker-compose.yml :
bash> docker-compose up -d

Vous disposez maintenant d’une machine configurée avec The Sleuth Kit.

3. Les premières étapes

Une fois le conteneur lancé, connectez-vous à la machine avec la commande :
bash> docker exec -ti tuto-thesleuthkit /bin/bash

Pour calculer la valeur de hachage d'un fichier :
bash> md5sum [nom_fichier]
Ici nous utilisons les fichiers texte.txt et fichier.dd.
bash> md5sum texte.txt
bash> md5sum fichier.dd

Pour afficher le volume des fichiers avec d'autres caractéristiques :
bash> img_stat texte.txt
bash> img_stat fichier.dd

4. Recherches plus en profondeur

Vous pouvez afficher l'ensemble des disques et leur stockage utilisé/restant.
bash> mmls [nom_image]

Cependant, fichier.dd a une extension dd ce qui signifie que c'est une image ou une partition d'un disque dur sous Linux. Celui présent dans la machine du conteneur docker est une image d'un disque dur virtuelle.
La commande mmsl affichera donc un code d'erreur. Elle ne fonctionne que pour les images de disque dur physique.

Vous pouvez afficher les détails et les statistiques du système de fichiers, y compris la disposition, les tailles et les étiquettes.
bash> fsstat fichier.dd

Vous pouvez afficher l'ensemble des répertoires et des fichiers présent dans l'image du disque dur.
bash> fls fichier.dd

L'option -o [nb_secteur] vous permet de sélectionner à partir de quel secteur vous voulez commencer à afficher les informations de l'image disque. Cependant dans le cas du fichier.dd, cette fonctionnalité est incompatible.
Voici ce que vous devriez obtenir :
bash> fsstat -o 11264 fichier.dd


Vous pouvez également récupérer les fichiers stockés dans fichier.dd
bash> tsk_recover -i raw -e fichier.dd ../TheSleuthKit/001

Aller ensuite dans le dossier 001
bash> cd 001

Vous pouvez voir tous les fichiers présents dans fichier.dd qui ont pu être récupérer. Le fichier réponse n'a pu être récupérer car il est dommagé. Cela est du à la copie partielle du disque dur virtuel effectué pour la réalisation de ce tutoriel.
bash> ls

Ceci conclut ce tutoriel. Pour quitter le docker entrez la commande :
bash> exit

Pour plus d'information sur le logiel The Sleuth Kit vous pouvez visiter le site internet : http://wiki.sleuthkit.org/index.php?title=TSK_Tool_Overview
