Tutorial HTTP Server

Objective

Our project involved launching an HTTP server in order to retrieve files through this web server. To do this, we had to create two images on docker, one to launch the server and the second to run the command to retrieve the file.
Documentation is available here : https://docs.python.org/3/library/http.server.html

1. Create the images

The first step was to create the images. To do so, we created a Dockerfile, which is a text configuration file containing a series of instructions for creating a custom Docker image. These instructions specify the steps required to build a Docker image from a base image, adding additional instructions.
Therefore, we had to create one Python image to create the server and one Ubuntu to execute the command.


2. Create the docker-compose file

This docker-compose.yml file defines two services (myubuntu and mypython) and a network (tuto_wget). The services are built from their respective Dockerfiles and are connected to the custom network with specific IP addresses.

3. In the command prompt, issue the command: docker-compose build

When you run this command, Docker Compose reads the configuration file, searches for the specified services, and executes the corresponding build instructions for each service.

4. In the command prompt, issue the command: docker-compose up -d

It creates and launches containers for all specified services using the corresponding images.
Each container is launched in detached mode, which means it runs in the background without looking at the terminal.



We can now see that the containers have been created with the images :

5. In the command prompt, issue the command: docker exec -ti myubuntu bash

This command is used to run a shell session inside the myubuntu container.

6. In the command prompt, issue the command: wget http://10.5.0.3

We can now make the http request, in order to access the wanted files.
